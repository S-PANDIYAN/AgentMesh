N+1 Query Problem (Line 112)

Database queries executed in a loop causing exponential data retrieval

HIGH

Impact: Potential timeout with large datasets, 10-100x slower than optimal

for (const user of users) {
  const posts = await db.query(`SELECT * FROM posts WHERE user_id = ${user.id}`);
  console.log(posts);
}
FIX: Use JOIN or Batch Query
Execute a single database query with JOIN instead of querying in a loop. This reduces database round-trips from N queries to 1.

Inefficient Loop - O(n²) Complexity (Line 234)

Nested loops with array lookups causing quadratic time complexity

MEDIUM

Impact: Execution time increases exponentially with data size (10x items = 100x slower)

for (let i = 0; i < items.length; i++) {
  for (let j = 0; j < items.length; j++) {
    if (items[i].id === items[j].id) {
      // process
    }
  }
}
FIX: Use Set or Map for lookups
Create a Set/Map from items first to achieve O(1) lookup time instead of O(n) array searches inside the loop.

Memory Leak - Event Listener Not Removed (Line 67)

Event listener added but never removed, causing memory accumulation

HIGH

Impact: Memory usage grows over time, potential application crash after prolonged use

FIX: Remove event listeners in cleanup
Always remove event listeners when they're no longer needed. Use cleanup functions in React useEffect or similar patterns.
